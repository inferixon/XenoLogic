# Oppgave 02 Dev Plan

Goal: prepare a Figma-ready site structure for a space-themed website that can be implemented with HTML and CSS in the next assignment.

Non-goals (for now): complex animations, unrealistic layouts, advanced app logic, framework-specific patterns.

## Core Decisions

- Choose one space-related angle.
- Decide early whether this should become a SPA or MPA.
- Keep every section easy to translate into semantic HTML.

## Required Frames

1. Desktop wireframe
2. Desktop final design
3. Mobile wireframe
4. Mobile final design

## Required Content Areas

- Header with logo and navigation
- Hero with title, short intro, and media
- At least two content sections or pages
- Footer with links and contact details

## Recommended Workflow

1. Write the chosen topic and section names in `Dev/content-outline.md`.
2. Build the desktop wireframe first using only structure.
3. Copy the wireframe and turn it into the desktop design.
4. Repeat both steps for mobile.
5. Check that the mobile layout stacks cleanly and stays realistic for HTML/CSS.
6. Store exported assets or reference files in `Assets/`.

## Coding Readiness Checklist

- Navigation labels match planned sections or pages.
- Hero content is short enough to fit on mobile.
- Cards, galleries, or media blocks can be built with flex or grid.
- The footer has a clear structure.
- Typography and color choices are limited and consistent.

## Definition of Done

- Four Figma frames are planned
- The theme is clearly space-related
- The structure includes the required sections
- Mobile and desktop both make sense
- The design can be implemented with plain HTML and CSS
