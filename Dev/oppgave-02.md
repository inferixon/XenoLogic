# Obligatorisk oppgave 02

Hei!

I denne oppgaven skal du planlegge og designe en enkel nettside i Figma. Du skal lage både wireframes og et ferdig design som senere skal brukes når du koder nettsiden i HTML og CSS i neste oppgave.


Det er viktig å huske at det du designer nå er det du senere skal lage med kode. Derfor bør du hele tiden tenke på hva som faktisk er mulig å bygge med HTML og CSS når du setter opp layout og design.


OBS: Temaet for neste oppgave er verdensrommet. Du kan selv velge hvilken del av temaet du vil fokusere på, for eksempel planeter, stjerner, romfartøy, astronauter eller noe annet relatert til verdensrommet som spill eller filmer om temaet.


Dette er også et godt tidspunkt å tenke på om du vil lage en SPA (Single Page Application) eller en MPA (Multi Page Application).


Målet med oppgaven er å lage en tydelig struktur og et design som kan brukes direkte når du senere skal kode nettsiden.

## Seksjoner som bør inkluderes

- Header: logo og navigasjon
- Hero-seksjon: intoduksjon om temaet med tittel, bilde/video og kort beskrivelse
- Minst to innholdseksjoner/innholdssider, for eksempel “Om universet”, “Planeter”, “Romfartøy” eller “Galleri”
- Footer: kontaktinformasjon, lenker og eventuelt sosiale medier (Fiktivt i dette tilfellet om verdensrommet)

## Steg 1: Lag en wireframe (desktop)


- Start med å lage en wireframe for desktop.
- Planlegg hvordan nettsiden skal struktureres ved å bruke enkle bokser og rammer som viser hvor innholdet skal plasseres.
- Fokuser kun på struktur og layout. Ikke bruk farger, bilder eller styling enda.

## Steg 2: Oppgrader wireframen til ferdig design (desktop)


- Lag en kopi av wireframen og bruk den som grunnlag for designet.
- Bytt ut placeholder-elementene med ekte innhold som passer til temaet verdensrommet. Det kan være tekst, bilder eller illustrasjoner.
- Velg en enkel fargepalett og tydelig typografi.
- Hold designet ryddig og oversiktlig, og tenk på hvordan det kan implementeres med HTML og CSS.

## Steg 3: Lag wireframe og design for mobil


- Gjenta steg 1 og 2, men for mobilvisning.
- Tenk gjennom hvordan layouten bør tilpasse seg en mindre skjerm, og hvordan innholdet kan stables eller omorganiseres.
- Ekstra utfordring (valgfritt)

## Hvis du ønsker en ekstra utfordring, kan du utforske følgende i Figma


- Bruk auto layout for å visualisere hvordan layouten kan bygges med flex i CSS
- Gi elementer navn som tilsvarer HTML-elementer, for eksempel header, nav, section eller footer
- Test prototyping i Figma for å lage en mer realistisk opplevelse av nettsiden
- Oppgaveinnlevering

Lever Figma-designfilen ved å trykke Share og levere lenken under copy.

Har lagt ved under en Figma Template som dere kan ta i bruk - siden som heter Oppgave 2 har 4 blanke templates med "riktige" størrelser som dere kan lage wireframe og design av.
