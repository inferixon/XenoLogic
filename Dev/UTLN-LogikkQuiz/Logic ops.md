# LogikkQuiz — Специфікація логічних операцій

**Мета**: Навчити дитину базовим логічним операціям через інтерактивні задачі норвезькою мовою.

**Статус реалізації**: 27 operation_types у коді, повне покриття практичних типів.

## Логічні операції (повний список)

### 1. Заперечення та еквівалентності
| Операція | Формула | Норвезький приклад | Статус |
|----------|---------|-------------------|--------|
| Негація твердження | ¬P | "Det er ikke sant at..." | ✅ `negation_introduction` |

### 2. Імплікація та дедукція
| Операція | Формула | Норвезький приклад | Статус |
|----------|---------|-------------------|--------|
| Імплікація | A → B | "Hvis A, så B" | ⏭️ Покрито через modus ponens/tollens |

| Modus tollens | A → B, ¬B ⊢ ¬A | "Hvis det regner, blir bakken våt. Bakken er ikke våt. Altså: det regner ikke." | ✅ `modus_tollens` |

## Прогресія складності (поточна реалізація рівні 1-3)

У коді зараз використовується 3 рівні складності (config `game.difficulty_level`), де складність впливає на якість дистракторів. Відповідність до початкової сітки:

| Рівень (код) | Що вкриває |
|--------------|------------|
| 1 | Базові імплікації, прості дистрактори (очевидні помилки) |
| 2 | Стандартні помилки умовиводу, додаються `insufficient_info`, складніші дистрактори |
| 3 | Тонкі підміни кванторів/модальності, ланцюги та пастки (affirming consequent / denying antecedent) |

Розширення до 8-рівневої градації з таблиці залишилось у беклозі.

## Пул дистракторів (хибні інференції)

### Для імплікації (A → B)
- **Affirming the consequent**: B ⊢ A (ХИБНО)
- **Denying the antecedent**: ¬A ⊢ ¬B (ХИБНО)
- **Додавання умови**: A ⊢ B ∧ C (C не в умові)

### Для modus ponens
- **Частковий висновок**: A ⊢ "щось схоже на B"
- **Інверсія сили**: A ⊢ "B сильніше/більше"

### Для modus tollens
- **Ігнорування заперечення**: ¬B ⊢ B
- **Помилкове узагальнення**: ¬B ⊢ "нічого не відомо" (коли відомо)

### Для кон'юнкції (A ∧ B)
- **Заперечення одного члена**: ⊢ ¬B
- **Перехід до або**: ⊢ A ∨ B
- **Порівняння/градація**: ⊢ "A сильніше за B"

### Для диз'юнкції (A ∨ B)
- **Помилкова ексклюзивність**: A ∨ B → A ⊕ B
- **Кон'юнктивна помилка**: A ∨ B → A ∧ B

### Для виключної диз'юнкції (A ⊕ B)
- **Інклюзивна підміна**: ⊢ A ∨ B (допускає обидва)
- **Подвійна істинність**: ⊢ A ∧ B
- **Третій стан**: "ні A, ні B" або "середній варіант"

### Для квантора ALLE
- **Зсув до NOEN**: ∀ → ∃
- **Заміна на INGEN**: ∀ → ¬∃

### Для квантора NOEN
- **Зсув до ALLE**: ∃ → ∀
- **Зсув до INGEN**: ∃ → ¬∃

### Для квантора INGEN
- **Зсув до NOEN**: ¬∃ → ∃
- **Ствердження протилежного класу**

### Для транзитивності
- **Обрив ланцюга**: A→B, B→C ⊢ тільки B
- **Зворотний ланцюг**: C ⊢ A (ХИБНО)

### Для винятків
- **Ігнорування винятку**: застосування правила без урахування "unntatt"
- **Перенесення винятку на весь клас**

### Для недостатності інформації
- **Вигаданий конкретний висновок**
- **Емоційно правдоподібний варіант**

### Для протиріччя
- **Прийняття обох тверджень як істинних**
- **Ігнорування одного з тверджень**

### Для закону виключеного третього
- **Розширення домену**: введення "середнього", "частково"
- **Уникнення вибору**: "не знаю", коли вибір примусовий

### Для темпоральних кванторів
- **Помилка моменту**: змішування "почнеться" з "вже почалося"
- **Зсув alltid ↔ noen ganger**

### Крос-дистрактори (для всіх типів)
- **Підміна предиката**: властивість/дія, якої немає в умові
- **Підміна класу**: заміна A на схожий, але інший клас
- **Лексична пастка**: красива, але логічно порожня фраза
- **Speech-act error**: сплутування заперечення повідомлення з описом фактів
- **Підміна істинності джерела**: "вони збрехали" ≠ "істина є протилежною"
